# Replication Recipe

> **For detailed expected outputs, known issues, and troubleshooting, see [REPLICATION_GUIDE.md](REPLICATION_GUIDE.md).**

A literal copy-pastable terminal session that reproduces every figure,
table, and result file in the manuscript from a fresh clone of this
repository. Each block is self-contained; you can rerun any block in
isolation if you only need to refresh one set of assets.

## Prerequisites

Before any commands below, ensure:

```bash
# Python 3.11–3.14 (we develop on 3.14)
python3 --version

# All pinned project deps installed
python3 -m venv .venv
source .venv/bin/activate
pip install -r supply_chain_research/requirements.txt

# Optional: webapp deps if you want to use the dashboard
pip install -r webapp/backend/requirements.txt
cd webapp/frontend && npm install && cd -
```

Hardware: the algorithm package itself runs on any modern laptop
(8 GB RAM minimum). The deep learning training (LSTM, PPO) benefits
from a GPU but is not required — a CPU-only fallback exists.

## Quick start (one command)

If `data/results/training_summary.json` is already populated
(downloaded from the Modal volume or generated by a prior training
run), the entire paper-asset bundle reproduces in ~3 minutes:

```bash
make paper-assets
```

This runs: figures + tables + thresholds + pytest suite,
all in sequence. Outputs land in `outputs/figures/`,
`outputs/tables/`, `outputs/managerial_thresholds.json`.

## Full reproduction from scratch

If you have **zero pre-computed results** on disk, follow the steps
below in order. Each step is idempotent — re-running it after a
crash picks up where it left off.

### Step 1 — Verify the test suite passes

```bash
make tests
```

Expected on the Python 3.14 audit environment: `488 passed, 5 skipped`
in about 205 seconds. This is your
first signal that the package is installed correctly. If anything
fails here, stop and resolve before going further.

### Step 2 — Reproduce the CVRPLIB Augerat-Set-A validation

This step is **fully self-contained** — no dependence on cloud
training. It downloads 27 standard benchmark instances from the
github mirror and produces the implementation-correctness table.

```bash
make cvrplib-validation
```

Expected output:

- `outputs/tables/cvrplib_validation.tex` — 27-row LaTeX table
- Console: mean gap-to-BKS in the 4–6 % band

### Step 3 — Run the full optimisation + ML pipeline

There are three options, ordered by speed and budget:

**Option 3a — Modal cloud GPU** (recommended; ~3 hours, ~$1.80):

```bash
modal run --detach cloud_training/modal_train.py
# (job runs detached; reconnect with: modal app logs supply-chain-ultimate-v3)
# When complete:
modal volume get sc-results-v3 / data/results --force
```

**Option 3b — Local CPU fallback** (free; ~6-10 hours):

```bash
python cloud_training/local_training_runner.py --component all
```

**Option 3c — Skip the pipeline, use the committed snapshot**:

If the ground-truth `data/results/*.{pkl,json,npy,pt}` files are
already present in your clone, skip to step 4. The downstream
figure / table generators will pick them up directly.

After this step, `data/results/` contains:

- `nsga2_all_results.pkl`, `nsga2_best_front.npy` (50-seed Pareto)
- `nsga3_all_results.pkl` (3-objective extension)
- `moead_all_results.pkl` (decomposition baseline)
- `best_lstm.pt`, `lstm_predictions.npy`, `lstm_actuals.npy`
- `ppo_small_final.pt`, `ppo_small_rewards.npy`
- `ppo_full_final.pt`, `ppo_full_rewards.npy`
- `ppo_baselines.json` ((R,s,S) + random)
- `mc_service_levels.npy` (DES 100 reps)
- `statistical_tests.json` (Friedman, Wilcoxon)
- `training_summary.json` (single source-of-truth aggregator)

### Step 4 — Render every paper figure

```bash
make figures
```

Expected output: `outputs/figures/fig{1..7}_*.png` and
`outputs/figures/supplementary/supp_fig{1,2}_*.png`. Plus the
3-objective NSGA-III projection figure:

```bash
python scripts/render_fig8_nsga3_projections.py
# -> outputs/figures/fig8_nsga3_projections.png
```

### Step 5 — Render every LaTeX table

```bash
make tables
```

Expected output:

- `outputs/tables/table1_parameters.tex`
- `outputs/tables/table2_algorithm_comparison.tex`
- `outputs/tables/table3_statistical_tests.tex`
- `outputs/tables/table4_resilience.tex`
- `outputs/tables/table5_ablation.tex`
- `outputs/tables/table6_sensitivity.tex`

Validate them syntactically:

```bash
make lint-tables
```

### Step 6 — Compute the managerial thresholds

```bash
make managerial-thresholds
```

Output: `outputs/managerial_thresholds.json` with:

- T1 — carbon price at which green-optimal beats cost-optimal
- T2 — fleet-mix shift breakeven
- T3 — days within a 30-day shock before service level breaches 95 %
- T4 — days of capacity loss the network sustains before SL < 95 %

### Step 7 — (Appendix) Run the secondary-network validation

```bash
python scripts/run_delhivery_secondary.py --seeds 20 --pop-size 300 --gen 80
```

Output:

- `data/results/delhivery_nsga2_all_results.pkl`
- `outputs/tables/secondary_network_validation.tex`

This is a cross-validation appendix for reviewers who ask whether
the primary-network results generalise to a different Indian
network. ~10 minutes on local CPU.

## One-shot total reproduction

Once `data/results/training_summary.json` exists (any of options
3a/3b/3c above), this single block reproduces *everything*:

```bash
# All paper assets — figures, tables, thresholds, tests
make paper-assets

# Bonus assets not in the make-target
python scripts/render_fig8_nsga3_projections.py
python scripts/run_cvrplib_benchmark.py
python scripts/run_delhivery_secondary.py --seeds 20

# Final sanity check
make lint-tables
```

After this completes the entire paper deliverable bundle is on disk.

## Debugging recipes

### "ImportError: No module named supply_chain_research"

You are not in the project root, or your venv is not activated.

```bash
cd /path/to/Supply-chain-main
source .venv/bin/activate
python -c "import supply_chain_research; print(supply_chain_research.__file__)"
```

### "make figures" reports SKIPPED for fig2–fig6

The figures depend on `data/results/*.{pkl,npy,pt}`. Run step 3
above (or its 3c snapshot fallback) and try again.

### Modal: "App is already stopped"

The previous app crashed or you ran `modal app stop`. Just relaunch:

```bash
modal run --detach cloud_training/modal_train.py
```

The skip-if-exists gates make this safe — no work is lost.

### CVRPLIB download fails

The github mirror may be temporarily unreachable. Cached files in
`data/external/cvrplib/` are reused, so partial coverage is OK.

### Tests fail after a clean `make clean`

`make clean` removes generated artefacts (figures, tables) but
intentionally not the `data/results/` snapshot. If you cleaned
those too, follow step 3 to regenerate.

## Reproducibility guarantees

Every script in this repo:

- Uses `numpy.random.SeedSequence` derivation through
  `supply_chain_research/utils/reproducibility.py`, so all RNGs are
  determined by a single project-wide seed (default 42).
- Pins exact dependency versions in `supply_chain_research/requirements.txt`
  and lock-step in the Modal `image.pip_install(...)` block.
- Writes results atomically through
  `supply_chain_research/utils/serialization.py` so crashes never
  leave half-written corrupted artefacts.

If two collaborators run this recipe on the same hardware with the
same seed, they should get bit-for-bit identical numeric outputs (to
within `1e-9` absolute tolerance for the floating-point pipeline,
documented in `audit_workspace/NUMERIC_BASELINE.json` and asserted by
`tests/test_regression_baseline.py`).
