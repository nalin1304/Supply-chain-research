## 5. PPO Policy Veto Mechanism

The Gymnasium environment enforces physical feasibility on PPO actions. When the agent requests more inventory than a warehouse has available, allocations are scaled proportionally:

```
scale = available_inventory / total_requested
actual_allocation = requested_allocation * scale
```

This preserves relative priority between customers while preventing impossible actions. The veto frequency decreases as training progresses (the agent learns feasibility constraints).
